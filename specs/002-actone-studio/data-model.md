# Data Model: ActOne Studio

**Branch**: `002-actone-studio` | **Date**: 2026-02-24

## Overview

ActOne Studio has two distinct type systems:

1. **Langium AST types** — generated by `langium-cli` from the `.langium` grammar. These represent the parsed structure of `.actone` files and live in `packages/langium`. They are generated code (Principle V) and must not be manually edited.

2. **Persistence types** — Drizzle ORM schemas for Supabase PostgreSQL tables. These represent project metadata, file storage, drafts, assets, snapshots, and analytics. They live in `packages/shared/src/db/schema.ts`.

The canonical shared type surface (`packages/shared/src/types/`) includes both re-exported AST types (for main-thread consumption) and all diagram, generation, publishing, and project types.

### 1.2 Projects Table

```typescript
export const projects = pgTable('projects', {
  id: uuid('id').primaryKey().defaultRandom(),
  userId: uuid('user_id').notNull(),  // references auth.users(id)
  title: text('title').notNull(),
  authorName: text('author_name'),
  genre: text('genre'),
  createdAt: timestamp('created_at', { withTimezone: true }).defaultNow().notNull(),
  modifiedAt: timestamp('modified_at', { withTimezone: true }).defaultNow().notNull(),
  version: text('version').default('1.0.0'),
  grammarVersion: text('grammar_version').default('2.0.0'),
  grammarFingerprint: text('grammar_fingerprint'),
  compositionMode: compositionModeEnum('composition_mode').default('merge').notNull(),
  lifecycleStage: lifecycleStageEnum('lifecycle_stage').default('concept').notNull(),
  publishingMode: publishingModeEnum('publishing_mode').default('text').notNull(),
}, (table) => [
  index('projects_user_id_idx').on(table.userId),
  pgPolicy('users_own_projects', {
    for: 'all',
    to: authenticatedRole,
    using: sql`(select auth.uid()) = ${table.userId}`,
    withCheck: sql`(select auth.uid()) = ${table.userId}`,
  }),
]);
```

**Fields**: id (PK), userId (FK to auth.users), title, authorName, genre, timestamps, version info, composition mode, lifecycle stage, publishing mode.

**RLS**: Users can only access their own projects.

### 1.3 Source Files Table

export const sourceFiles = pgTable('source_files', {

**Fields**: id (PK), projectId (FK), filePath (e.g., 'story.actone'), content (file text), isEntry (main file flag), timestamps.

**Constraints**: Unique (projectId, filePath). Cascade delete with project.

## 5. Langium AST Types (Generated)

The following types are generated by `langium-cli` from the `.langium` grammar file. They live in `packages/langium/src/generated/ast.ts` and must never be manually edited.

### 5.1 Root Type

- **Story**: `{ $type: 'Story'; name: string; elements: StoryElement[]; }`
- **StoryElement**: `CharacterDef | WorldDef | ThemeDef | TimelineDef | SceneDef | PlotDef | InteractionDef | GenerateBlock`

### 5.2 Element Types

- **CharacterDef**: name, properties (nature, bio, role, voice, personality, quirks, goals, conflicts, strengths, flaws, relationships, arc, symbols, secret, notes)
- **WorldDef**: name, properties (period, sensory, locations, rules)
- **ThemeDef**: name, properties (statement, motifs, counter, tension)
- **TimelineDef**: name, properties (structure, span, layers)
- **SceneDef**: name, properties (type, location, pov, layer, participants, atmosphere, objective, trigger, transition)
- **PlotDef**: name, properties (conflict_type, resolution_pattern, beats, subplots)
- **InteractionDef**: name, properties (participants, pattern, style_mix, subtext, power_dynamic, emotional_arc)
- **GenerateBlock**: properties (temperature, max_tokens, continuity_loss, style_bleed, genre, tone, tense, default_pov, pacing, chapter_breaks)

### 5.3 Sub-Types

- **PersonalityTrait**: `{ name: string; value: number; }` (value 0–100)
- **GoalEntry**: `{ goal: string; priority: 'Primary' | 'Secondary' | 'Hidden'; stakes: string; }`
- **RelationshipEntry**: `{ to: Reference<CharacterDef>; weight: number; label: string; history: string; dynamic: boolean; }` (weight −100 to +100)
- **ArcProp**: `{ description: string; start: string; catalyst: string; midpoint: string; turning_point: string; end: string; }`
- **LocationEntry**: `{ name: string; description: string; atmosphere: MoodEntry[]; connects_to: Reference<LocationEntry>[]; }`
- **MoodEntry**: `{ name: string; value: number; }` (value 0–100)
- **RuleEntry**: `{ rule: string; category: 'Physical' | 'Social' | 'Metaphysical' | 'Narrative' | 'Psychological'; }`
- **TimelineLayer**: `{ name: string; description: string; period: string; }`
- **PlotBeat**: `{ beat: string; type: BeatType; act: number; }`
- **BeatType**: `'Setup' | 'Inciting' | 'Rising' | 'Midpoint' | 'Complication' | 'Crisis' | 'Climax' | 'Falling' | 'Resolution' | 'Denouement'`
- **SubplotDef**: `{ name: string; description: string; beats: string[]; converges_at: Reference<SceneDef>; }`
- **StyleMixEntry**: `{ character: Reference<CharacterDef>; value: number; }`

### 5.4 Enum Types

- **CharacterNature**: `'Human' | 'Force' | 'Concept' | 'Animal' | 'Spirit' | 'Collective' | 'Environment'`
- **SceneType**: `'Action' | 'Dialogue' | 'Reflection' | 'Montage' | 'Revelation' | 'Confrontation' | 'Transition' | 'Climax'`
- **TimelineStructure**: `'Linear' | 'Nonlinear' | 'Parallel' | 'Collapsed' | 'Cyclical' | 'Reverse'`
- **TransitionType**: `'Cut' | 'Dissolve' | 'Flashback' | 'FlashForward' | 'Parallel' | 'Smash' | 'Fade' | 'Montage'`
- **ConflictType**: `'Interpersonal' | 'Internal' | 'Intrapsychic' | 'Societal' | 'Environmental' | 'Cosmic' | 'Existential' | 'Technological'`
- **ResolutionPattern**: `'Transformative' | 'Tragic' | 'Redemptive' | 'Ambiguous' | 'Cyclical' | 'Pyrrhic' | 'Transcendent'`
- **Tense**: `'Past' | 'Present' | 'Future'`
- **POV**: `'FirstPerson' | 'SecondPerson' | 'ThirdLimited' | 'ThirdOmniscient'`
- **Pacing**: `'Slow' | 'Measured' | 'Moderate' | 'Brisk' | 'Accelerating'`
- **GoalPriority**: `'Primary' | 'Secondary' | 'Hidden'`
- **RuleCategory**: `'Physical' | 'Social' | 'Metaphysical' | 'Narrative' | 'Psychological'`

## 7. Entity Relationship Summary

```
projects 1──* source_files
projects 1──* snapshots 1──* snapshot_files
projects 1──* assets
projects 1──* analytics_snapshots
projects 1──* draft_versions

auth.users 1──* projects (via user_id)
```

All child tables cascade delete when the parent project is deleted. All tables enforce user-scoped RLS via the project's `user_id`.
